Axe 1 — Class Imbalance

المشكلة والمقاييس

المفهوم (Concept)

  • • Class Imbalance: Exemple: 70% Positif → accuracy fausse
  • • F1-macro: Métrique principale du projet
  • • F1 par classe: Positive / Negative / Neutral

ما يجب فهمه (À comprendre)

  • → Pourquoi accuracy est trompeuse avec déséquilibre
  • → Trade-off Precision / Recall
  • → AUC-PR meilleur que AUC-ROC sur données déséquilibrées
  • → G-mean: racine carrée de la moyenne géométrique des Recalls
  • → Confusion Matrix à 3 classes: visualisation des erreurs

Axe 2 — Transformers, BERT & Fine-tuning

الترانسفورمرز والتعديل الدقيق

Architecture BERT

  • • Encoder bidirectionnel
  • • Attention Mechanism
  • • Tokenization spécifique
  • • Token [CLS]: vecteur 768 dimensions

DziriBERT & Configuration

  • • Spécialisé pour la darija algérienne
  • • 20M tokens, support Arabizi
  • • epochs=5, lr=2e-5, batch=16
  • • seed=42, AdamW — Ne pas changer!

Axe 3 — Loss Functions المعدلة

Loss Function Description الوصف
CrossEntropy standard Baseline sans pondération الخسارة القياسية
ClassWeighting wi = total / (nb_classes × nb_class_i) ترجيح حسب حجم الفئة
Focal Loss Focus sur exemples difficiles avec γ التركيز على الأمثلة الصعبة
γ = 2 vs γ = 1 γ=0 → CrossEntropy | γ=2 → plus agressif معامل التركيز
Combinaison CW + FL Class Weighting + Focal Loss combinés الجمع بين الطريقتين

Axe 4 — Embeddings & SMOTE

  • [CLS] Embeddings: Extraction vecteur 768 de DziriBERT sans tête de classification
  • SMOTE: Génération de points synthétiques par interpolation K-NN
  • ADASYN: Extension SMOTE adaptative selon densité
  • Test: Rééquilibrage total vs partiel — comparer F1-macro

Axe 5 — Back-Translation

الترجمة العكسية وتوليد البيانات

  • Processus: Tweet → Français → Arabe = paraphrase naturelle
  • Helsinki-NLP: Modèle AR↔FR open-source sur HuggingFace
  • Filtrage cosinus: 0.5–0.85 (sinon rejet)
  • Taux: 100% / 50% / 20% de la classe minoritaire
